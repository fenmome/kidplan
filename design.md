你是一名专注于自托管轻量应用的全栈开发工程师，请严格按照以下需求，开发一套完整的「儿童周日程管理系统」，采用前后端分离架构，输出所有可直接运行的代码与部署配置。

## 项目概述

- 项目代号：kidplan
- 使用场景：个人私有使用，单管理员账号，用于孩子的每周日程规划与提醒
- 部署方式：Docker Compose 部署，线上服务器运行

## 技术栈强制要求

### 后端

- 语言：Python 3.11
- 框架：FastAPI + SQLAlchemy 2.0 + APScheduler + PyJWT
- 数据库：SQLite，数据库文件路径可配置
- 接口规范：RESTful风格，统一响应格式，自动生成/docs接口文档
- 认证：JWT令牌认证，登录接口获取token，所有业务接口校验token

### 前端

- 框架：Vue 3 + Vite + Element Plus + Pinia + Axios
- 构建：生产环境打包为静态文件
- 适配：PC端优先，兼容移动端浏览器
- 路由：Vue Router，未登录自动跳转登录页

### 部署

- 根目录包含 backend/、frontend/ 两个独立文件夹
- 根目录提供 docker-compose.yml
- 前端Dockerfile：多阶段构建，最终用nginx托管静态资源，同时配置反向代理转发/api到后端
- 后端Dockerfile：基于python:3.11-slim镜像
- 所有配置通过环境变量传入，禁止硬编码

## 核心功能需求

### 后端功能

1. 日程管理接口
   - 日程字段：id、标题、开始时间、结束时间、所属星期（1-7）、备注、是否启用提醒、提前提醒分钟数、创建时间、更新时间
   - 接口：新增、编辑、删除、查询单周所有日程、复制本周日程到下周
   - 按周维度管理日程，支持循环复用

2. 认证接口
   - 管理员登录接口，校验用户名密码，返回JWT token
   - 账号密码通过环境变量配置，仅单管理员账号

3. 定时提醒模块
   - APScheduler进程内调度，每分钟轮询检测即将开始的日程
   - 支持全局默认提前提醒时长（默认10分钟），单条日程可单独覆盖
   - 同时兼容钉钉自定义机器人、飞书自定义机器人Webhook，自动适配官方消息格式
   - 消息内容：【日程提醒】标题 + 开始时间 + 备注
   - 记录提醒发送日志（时间、日程id、发送状态、返回结果）

4. 系统配置接口
   - 可查询当前系统配置项（提醒时长、通知渠道状态等）

### 前端功能

1. 登录页
   - 简单账号密码登录表单，错误提示，登录成功跳转首页

2. 周视图主页面
   - 核心展示：横向表头为周一至周日，纵向时间轴为6:00-22:00，按小时划分
   - 日程块按时间位置渲染在对应日期列，不同颜色区分是否启用提醒
   - 点击空白时间格：弹出新增日程弹窗
   - 点击已有日程块：弹出编辑/删除弹窗
   - 顶部操作栏：切换周、复制本周到下周、全局设置按钮

3. 全局设置弹窗
   - 配置钉钉Webhook地址、飞书Webhook地址、默认提前提醒分钟数
   - 配置保存后即时生效

4. 提醒日志页面
   - 列表展示近期提醒发送记录，包含时间、日程标题、发送状态
